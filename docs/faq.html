FAQ

Q. '인증에 실패했습니다.' 에러는 어떤 경우에 발생하나요?​

CASE 1: 유효하지 않은 액세스 토큰​

{
    "errorMessage": "result is false : Authentication failed. (인증에 실패했습니다.)",
    "errorCode": "024"
}

Authorization 헤더에 'Bearer {엑세스 토큰}' 형태로 엑세스 토큰을 전달했지만, 엑세스 토큰이 만료되었거나 유효하지 않은 경우에 나타날 수 있는 오류 코드 입니다.

엑세스 토큰을 새로 발급 받아서 다시 시도해 보시기 바랍니다.

'네이버 로그인 API 명세' 바로가기

'튜토리얼 > Web 애플리케이션' 바로가기

CASE 2: 권한 부여 전 발급받은 액세스 토큰​

{
    "errorMessage": "GatewayException : Authentication failed. (인증에 실패했습니다.)", 
    "errorCode": "024" 
}

API 사용 권한을 부여해 드리기 전에 '서비스 동의'를 하셨을 때 발생하는 현상입니다.

'서비스 동의 철회' 후 다시 '서비스 동의' 하기 위해 아래 절차를 진행해 주세요.

  1. www.naver.com 에 로그인한 상태에서 네이버 '프로필 영역'이나 '네이버ID'를 클릭하시고,
  2. '이력관리' 클릭, '연결된 서비스 관리' 옆 전체보기 클릭하시면 NAVER developers에서 등록하신 Application 이름을 찾으실 수 있습니다.
  3. Application 이름을 클릭하셔서 '서비스 동의 철회' 를 실행하시고
  4. 네이버 로그인과 액세스 토큰 발급 과정을 다시 진행해 보시기 바랍니다.

Q. API 응답 필드의 의미는 어떻게 알 수 있나요?​

가이드의 'API' 메뉴를 펼치면 '계정API' 를 시작으로 API 요청/응답 명세를 보실 수 있습니다.

Responses 항목에서 '200 OK' 부분을 클릭해서 펼치면 응답 필드 각각의 의미를 확인하실 수 있습니다.

Q. 관리계정 하위 광고계정의 캠페인/광고그룹/소재를 관리할 수 있나요?​

A. 광고계정 멤버가 아니라도 관리계정의 권한레벨에 따라 하위 캠페인/광고그룹/소재 조회,수정,삭제 API를 사용할 수 있습니다.

예를 들어, 관리계정 123에 '운영조회' 또는 '운영관리', '마스터' 권한레벨을 가지고 있는 네이버 아이디로 관리계정 123의 하위 광고계정 456의 캠페인을 조회할 수 있습니다.

AccessManagerAccountNo 헤더로 관리계정Id를 전달하면 됩니다. 자세한 내용은 아래 바로가기를 클릭해서 확인해 보세요.

'관리계정으로 접속하여 호출' 바로가기
